# Laravel 11 AI Instruction Protocol

Follow this skill when working under a Laravel 11 project that uses the user's custom protocol.

## Enforce the execution gate

- Treat `Go!` as the only authorization to implement.
- Before `Go!`, only read, analyze, confirm understanding, and ask clarifying questions when required.
- Before `Go!`, do not generate code, edit files, or perform implementation work.
- After `Go!`, execute the requested objectives exactly as instructed and do not add unrequested alternatives.

## Use the protocol as the source of truth

- Follow the detailed rules in [references/laravel-protocol.md](references/laravel-protocol.md).
- Prefer the user's protocol over Laravel defaults when they conflict.
- Call out meaningful conflicts only when they create risk or block implementation.

## Apply the implementation workflow

1. Read the request and determine whether `Go!` has been given.
2. If `Go!` has not been given, limit the response to understanding, analysis, and clarifications.
3. If `Go!` has been given, identify whether the task creates new files or modifies existing ones.
4. Use Artisan and terminal-based generation for Laravel-created files whenever the protocol requires it.
5. Keep controllers slim, models data-focused, routes synchronized, and Blade presentation-only.
6. Use jQuery + AJAX and JSON responses for form workflows that submit data.
7. Deliver output in the format required by the protocol: full files for new files, modified blocks only for existing files.

## Apply file-generation rules

- Use Artisan commands for migrations, models, controllers, requests, seeders, and factories.
- Never handcraft files that should be generated by Artisan.
- When instructing or reporting file creation, include the exact Artisan command and the resulting file path.

## Apply delivery rules

- Provide full contents for newly created files.
- Provide only the modified blocks for updates to existing files, unless the user explicitly asks for the full file.
- Keep solutions simple, direct, and optimized.
- Deliver only what the user asked for.
